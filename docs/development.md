# Разработка и качество

## Установка

```bash
uv sync
```

## Проверки

Формат и линтинг:

```bash
uv run ruff check
```

Тесты:

```bash
uv run pytest
```

## Текущий статус тестов

В текущем состоянии репозитория автотесты отсутствуют (`collected 0 items`).

Рекомендуемое направление:

- добавить unit-тесты для `domain` (цель: 100% покрытия домена по правилам проекта);
- добавить unit/integration тесты для application/infrastructure по контрактам портов.

## Стиль кода

- Соблюдать гексагональную архитектуру и изоляцию слоев.
- Для новых публичных методов в `domain/application/ports/presentation` писать осмысленные docstring на русском языке.
- В SQL использовать параметризованные запросы.
- Не добавлять неиспользуемые импорты.

## Рекомендованный workflow

1. Внести изменения в одном слое и проверить границы портов.
2. Запустить `ruff`.
3. Запустить `pytest`.
4. Обновить документацию при изменении контрактов/поведения.
